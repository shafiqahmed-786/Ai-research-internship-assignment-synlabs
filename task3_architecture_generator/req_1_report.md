# Report for req_1

## Input requirement

Build an e-commerce product listing and checkout system where users can browse products, add to cart, and checkout using credit/debit or wallet. Admins must be able to add products, update inventory, and view orders. The system should support order history and basic analytics for top-selling products.

## Modules

- **Auth**: User registration, login, and session management.

- **Catalog**: Product listing, search, filters, and product details.

- **Cart**: Temporary cart storage and manipulation.

- **Checkout**: Order creation, payment integration, and order confirmation.

- **Admin**: Product CRUD, inventory updates, and order management.

- **Analytics**: Aggregate sales, top-selling products and reporting.


## Schema

### Table: sources

- **id** (uuid) — PK: Source id
- **name** (varchar(255)): Source name
- **type** (varchar(50)): rss/twitter/api

### Table: content

- **id** (uuid) — PK: Content id
- **source_id** (uuid): FK to sources
- **raw_text** (text) — nullable: Original text
- **topic** (varchar(100)) — nullable: Assigned topic
- **ingested_at** (timestamp): Ingest time

## Pseudocode

### add_to_cart

- Validate user session or allow guest cart

- Fetch product by id

- Check inventory >= requested quantity

- Create or update cart item with quantity

### checkout

- Validate cart is not empty

- Calculate totals and taxes

- Validate payment info and call payment gateway

- On success: create order record, decrement product inventory, send confirmation email
